# Day 22 Notes

## 1. Difference between git add and git commit

git add moves changes from the working directory to the staging area.

git commit saves the staged changes permanently into the repository history

---

## 2. What does the staging area do?

The staging area allows developers to select which changes should go into the next commit. It helps create clean and meaningful commits instead of committing everything.

---

## 3. What information does git log show?

git log shows commit history including:
- Commit ID
- Author name
- Date
- Commit message

---

## 4. What is the .git folder?

The .git folder stores all repository data such as commits, branches, configuration, and history. If this folder is deleted, the project will no longer be a Git repository and all version history will be lost.

---

## 5. Difference between working directory, staging area, and repository

Working Directory: where you edit files.

Staging Area: place where changes are prepared for commit.

Repository: database where Git permanently stores commits.
